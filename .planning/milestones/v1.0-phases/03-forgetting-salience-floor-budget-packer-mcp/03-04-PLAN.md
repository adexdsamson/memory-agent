## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → MCP tool layer | user_id crosses this boundary as an explicit tool argument |
| MCP tool layer → MemoryEngine | tool functions delegate immediately; no business logic in tool layer |
| MCP server (stdio) → caller process | stdio transport is local-only; no network exposure in MVP |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04-01 | Tampering | MCP user_id isolation — tool call for user A must not read/mutate user B | mitigate | user_id is explicit, required, non-defaulted on every tool (D3-14). Engine enforces scope isolation on every read/write (D-02/D-03). Test: expand called with wrong user_id returns None (scope check proven). consolidate passes user_id to engine so consolidation is scoped. |
| T-03-04-02 | Tampering | MCP forget — caller evicts another user's record by guessing record_id | mitigate | engine.forget() fetches record and verifies record.user_id == caller's user_id; raises ValueError on mismatch. FastMCP propagates error as tool error to client. |
| T-03-04-03 | Tampering | MCP remember content injection — content field contains MCP-framing characters | accept | FastMCP validates tool args with Pydantic; content is typed str; stored as parameterized SQL in T1 (no interpolation). No new surface. |
| T-03-04-04 | Tampering | MCP recall budget manipulation — caller sets budget=0 to get empty results | accept | Empty packed result is a valid (if unhelpful) response; caller controls their own recall budget. No data leakage possible from a filtered result. |
| T-03-04-05 | Repudiation | No authentication on stdio MCP tools | accept | MVP: stdio is local-only; explicit user_id arg provides identity per call. HTTP/SSE transport + auth → post-MVP (deferred per 03-CONTEXT.md). Document this limitation. |
| T-03-04-06 | Tampering | consolidate user isolation via MCP | mitigate | consolidate tool passes user_id to engine.consolidate(user_id=...) which scopes pipeline to that user's staged items — a user cannot trigger consolidation of another user's data. Tested by test_mcp_consolidate_passes_user_id. |
</threat_model>
