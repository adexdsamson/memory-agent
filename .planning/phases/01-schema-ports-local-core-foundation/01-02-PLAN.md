## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-04 | Information Disclosure | `SqliteT1.vector_search` — cross-scope read if user_id predicate missing | mitigate | `user_id` is non-defaulted after `*` in both the Protocol and the adapter; the WHERE clause always includes `AND r.user_id = :user_id AND r.valid_until IS NULL`. `test_recall_does_not_leak_across_users` asserts this at the test layer. |
| T-1-05 | Tampering | `SqliteT1.update` — SQL injection via dynamic field names | mitigate | `update()` builds the SET clause from a whitelist of allowed column names (all MemoryRecord field names), not from raw caller-supplied strings. Parameterized values. |
| T-1-06 | Information Disclosure | `LocalFS.get(ref)` — path traversal if session_id contains `..` | mitigate | `LocalFS.get` parses the `t0://session_id/offset` ref and constructs `self._base / f"{session_id}.jsonl"`. Validate that session_id contains only alphanumeric characters, hyphens, and underscores before path construction. Raise `ValueError` on invalid session_id. |
