CRITICAL — DEMO-03 entity resolution requirement:
  StubEmbedder uses sha256(content.encode()) → deterministic float vector.
  Two DIFFERENT content strings produce distance > ENTITY_MAX_DISTANCE (0.548) → entity resolution does NOT fire.
  Identical content strings produce distance = 0 → entity resolution fires → contradiction judge runs.
  Therefore DEMO-03 MUST use the same content string for both remember() calls.

CRITICAL — DEMO-03 content must NOT trigger safety keywords (no "allerg", "anaphyl", etc.) or CONS-08 blocks auto-supersession.
  "spicy food preference item 0" contains no safety keywords — safe to use.

CRITICAL — DEMO-02 sequencing:
  1. remember() → staging queue (in-memory)
  2. consolidate() → flushes staging queue to confirmed T1 (SQLite file)
  3. eng._t1.close() → WAL checkpoint + db close
  4. build_engine(same_cfg) → new engine over same SQLite file
  5. recall() → finds the confirmed record
  Without step 2 before step 3, provisional records in the staging queue are lost on engine close.
