Key DDL differences from SQLite (04-PATTERNS.md PostgresT1 section + 04-RESEARCH.md Pattern 5):
  SQLite types → Postgres types:
    INTEGER (protected/provisional) → BOOLEAN
    TEXT (valid_from/valid_until/last_accessed/created_at) → TIMESTAMPTZ
    TEXT (keywords/source_refs/graph_edges) → JSONB
    REAL (salience/confidence) → FLOAT

  Two-table design (mirrors sqlite vec_t1 virtual table):
    t1_records — main record columns (no embedding column)
    t1_vectors — (record_id TEXT PK FK t1_records.id, embedding vector(dim))
    HNSW index on t1_vectors.embedding
    Partial index on t1_records (user_id, agent_id) WHERE valid_until IS NULL

  psycopg3 differences from aiosqlite:
    %s placeholders (not ?)
    await conn.execute() returns a cursor with .rowcount
    await conn.commit() for autocommit=False connections
    async with conn.transaction(): context manager for atomic blocks
    JSON/JSONB columns returned as dict/list natively (no json.loads)
    BOOLEAN returned as Python bool natively (no int() cast)
    TIMESTAMPTZ returned as Python datetime with tzinfo (needs timezone-aware comparison)

  _ALLOWED_COLUMNS frozenset: copy verbatim from sqlite_t1.py lines 36-62
  _make_record() row factory: psycopg3 cursor rows are tuples; build dict manually

Critical implementation notes (04-RESEARCH.md Pitfalls 1 and 2):
  Pitfall 1: call register_vector_async(conn) IMMEDIATELY after connect, before any query
  Pitfall 2: SET hnsw.iterative_scan = 'strict_order' at session level before every KNN query
  Security: user_id = %s predicate on EVERY SELECT, UPDATE (scope isolation)
  Security: _ALLOWED_COLUMNS whitelist check in update_fields() before column interpolation

pgvector version guard (CLAUDE.md pin >= 0.8.2 / CVE-2026-3172):
  After _create_schema(), before returning from open(), execute:
    SELECT extversion FROM pg_extension WHERE extname='vector'
  Parse the version string (e.g. "0.8.2") and compare as a tuple of ints.
  If version < (0, 8, 2): raise RuntimeError("pgvector extension version {ver} < 0.8.2 (CVE-2026-3172). Upgrade pgvector.")
  This fires at adapter open time so the error is clear and early.

    supersede(self, old_id: str, new_record: MemoryRecord, embedding: list[float]) -> None:
      async with self._conn.transaction():
        result = await self._conn.execute("UPDATE t1_records SET valid_until=NOW(), superseded_by=%s WHERE id=%s AND user_id=%s", (new_record.id, old_id, new_record.user_id))
        if result.rowcount != 1:
          raise ValueError(f"supersede(): old_id={old_id!r} not found or user_id mismatch")
        INSERT new record into t1_records
        INSERT into t1_vectors

    vector_search(self, query_vec: list[float], k: int, *, user_id: str, agent_id: str | None = None) -> list[tuple[str, float]]:

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user_id param → Postgres queries | user_id is parameterized (%s); SQL injection impossible for param values; column names in update_fields() are whitelisted |
| vector_search result → application | Returns (record_id, distance) pairs only; no raw embedding data returned to caller |
| supersede() transaction | Atomic; rollback on failure prevents partial state (old retired, new not created) |
| pgvector extension version → PostgresT1.open() | Version guard raises RuntimeError if installed pgvector < 0.8.2 (CVE-2026-3172) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05-01 | Tampering | SQL injection via update_fields column names | mitigate | _ALLOWED_COLUMNS frozenset check raises ValueError before any column name interpolation — same mitigation as sqlite_t1.py |
| T-04-05-02 | Information Disclosure | Cross-user vector search | mitigate | WHERE user_id = %s predicate on every SELECT and vector_search; scope isolation test in conformance suite asserts this |
| T-04-05-03 | Tampering | Hard-delete via delete_vector | mitigate | delete_vector() deletes from t1_vectors only (ghost-record vector removal); t1_records row preserved with valid_until set. No DELETE FROM t1_records — FORG-04 |
| T-04-05-04 | Denial of Service | HNSW scan without iterative_scan | mitigate | SET hnsw.iterative_scan = 'strict_order' before every KNN query — Pitfall 2; ensures k results returned even with valid_until IS NULL filter |
| T-04-05-05 | Tampering | supersede() partial transaction | mitigate | async with self._conn.transaction() context manager rolls back atomically on exception — rowcount check prevents orphaned superseded_by pointers |
| T-04-05-06 | Tampering | CVE-2026-3172 pgvector parallel HNSW buffer overflow | mitigate | _check_pgvector_version() in open() raises RuntimeError if ext version < 0.8.2; testcontainers uses pgvector/pgvector:pg16 (ships >= 0.8.2) |
